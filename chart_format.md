# DDR 譜面 JSON フォーマット仕様

## 概要

`iwu.json` はDDR（Dance Dance Revolution）の譜面データをJSON形式で表現したものです。
元となるフォーマットはStepMania（SM/SSC）形式に準拠しています。

---

## データ構造

```
notes
 └─ measure（小節）の配列
      └─ beat group（拍グループ）の配列
           └─ subdivision（細分）の配列（4要素固定）
                └─ "LDUR" 文字列（4文字）
```

```json
{
    "notes": [
        [                          // 小節 (measure)
            [                      // 拍グループ (beat group = 1拍分)
                "0000",            // その拍の 1つ目の16分音符（拍頭）
                "0000",            // その拍の 2つ目の16分音符
                "0000",            // その拍の 3つ目の16分音符
                "0000"             // その拍の 4つ目の16分音符
            ],
            ...                    // beat group が4つで1小節（4/4拍子）
        ],
        ...
    ]
}
```

---

## 4文字の意味（パネル対応）

各 subdivision は4文字の文字列で、それぞれDDRの矢印パネルに対応しています。

| 位置 | 方向 |
|------|------|
| 1文字目 | ← 左 (Left) |
| 2文字目 | ↓ 下 (Down) |
| 3文字目 | ↑ 上 (Up) |
| 4文字目 | → 右 (Right) |

---

## ノート値の意味

| 値 | 種類 | 説明 |
|----|------|------|
| `0` | 空 | ノートなし |
| `1` | タップノート | 通常の矢印（踏むだけ） |
| `2` | ホールド開始 | 長押しの始点 |
| `3` | ホールド終了 | 長押しの終点 |
| `4` | ロールノート開始 | 連続して踏み続ける矢印の始点 |

---

## 具体例

### タップノートの例

```json
["0001", "0000", "1000", "0000"]
```

| インデックス | 文字列 | 意味 |
|-------------|--------|------|
| [0] 拍頭（1つ目の16分） | `"0001"` | → 右 矢印（タップ） |
| [1] 2つ目の16分 | `"0000"` | なし |
| [2] 3つ目の16分 | `"1000"` | ← 左 矢印（タップ） |
| [3] 4つ目の16分 | `"0000"` | なし |

### ホールドノートの例

```json
["2000", "0000", "3000", "0000"]
```

| インデックス | 文字列 | 意味 |
|-------------|--------|------|
| [0] 拍頭（1つ目の16分） | `"2000"` | ← 左 ホールド開始 |
| [1] 2つ目の16分 | `"0000"` | なし |
| [2] 3つ目の16分 | `"3000"` | ← 左 ホールド終了 |
| [3] 4つ目の16分 | `"0000"` | なし |

### 複数パネル同時押しの例

```json
["1001", "0000", "0000", "0000"]
```

| インデックス | 文字列 | 意味 |
|-------------|--------|------|
| [0] 拍頭（1つ目の16分） | `"1001"` | ← 左 と → 右 の同時押し |

---

## subdivision と音符の対応

各 beat group 内の4要素は、**その拍内の16分音符4つ**を順番に表します。
解像度は 16分音符固定（1小節 = 4拍 × 4細分 = 16行）。

| インデックス | 拍内の位置 | BPM160時の間隔 |
|-------------|-----------|---------------|
| [0] | 拍頭（1つ目の16分） | 0 ms |
| [1] | 2つ目の16分 | +93.75 ms |
| [2] | 3つ目の16分 | +187.5 ms |
| [3] | 4つ目の16分 | +281.25 ms |

> **注意**：かつて「4分・8分・16分・32分音符」と表現していたが誤り。  
> 正しくは同一解像度（16分）の連続した4ステップ。
